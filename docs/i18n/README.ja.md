# add-reasoning-to-prs

**自己文書化するPR。** 変更の *理由* — 決定、トレードオフ、前提、既知の制約 — を、すべてのプルリクエストに自動で書き込む [Claude Code](https://docs.claude.com/en/docs/claude-code) のフックです。あなた自身のエージェントが、あなた自身のマシン上で作成します。

AIがコードを書く。しかし *理由* は誰も書き残さない。3週間もすれば、あなたのGit履歴は、
名目上はあなたのものでも実際には自分で下したことのない決定の山になる — diffは *何が*
変わったかを示し、`git blame` は *誰が* 変えたかを示すが、その理由はただ… 消えている。

これは、それを根本から直すフックです。エージェントがPRを開く（あるいはデフォルトブランチに
コミットを積む）直前に、短い **「理由」ブロック** — 下した決定、天秤にかけたトレードオフ、
前提としたこと、そして意図的に残したこと — を説明文に直接書き込みます。実際のセッションから。
diffからではなく。

## インストール

コマンドひとつ:

```sh
npx add-reasoning-to-prs
```

これで、自己完結型・依存ゼロのフックが安定した場所にコピーされ、あなたのClaude Code設定
（`~/.claude/settings.json`）に `PreToolUse` フックとして登録されます。ビルド手順も、アカウ
ントも、設定も不要です。

または、マーケットプレイスから **Claude Code プラグイン** をインストールしてください（推奨 —
プラグインマニフェストからフックを登録し、プロジェクト設定に触れることなくプラグインと一緒に
更新されます）。

> **要件:** Claude Code と Node.js ≥ 22.18。現時点ではClaude Codeのみ対応 — Cursor と Codex
> が次の予定です。

## 何をするのか

| | |
|---|---|
| **PR作成時に「理由」ブロックを自動生成** | エージェントが「理由」ブロックなしで `gh pr create` を実行した（またはデフォルトブランチにコミットした）とき、フックはまずブロックを書くよう促し、その後コマンドを再実行させます。一度インストールすれば、手作業はゼロです。 |
| **前向き専用 — diffを言い換えない** | diffでは *示せない* もの、つまり推論と、承知の上で取ったリスクを捉えます。コードからは読み取れない一点を先頭に置き、「Xをリファクタリング、Yを改善」といった水増しは決してしません。 |
| **ローカル、あなた自身のサブスクリプション、アカウント不要** | ブロックはセッション内で、あなた自身のエージェントが作成します。サーバーへの往復もなく、何も保存されず、ソースコードがマシンから出ることはありません。 |
| **決してでっち上げない** | すべての行が、セッション内の実際の決定にたどれなければなりません。エージェントが実際には検討しなかったなら、ブロックは追加されません — 定型的な変更に対しては、空のブロックこそが正しい答えです。 |

## これが **しない** こと

- **レビューボットではありません。** コードを採点したり、PRにスコアを付けたり、マージをブロックしたりはしません。diffの *上* に、レビューと並んで位置します — 文脈を加えるのであって、判定はしません。
- **図でも、Wikiでも、ナレッジグラフでもありません。** 閲覧するものも、同期を保つものもありません。ただ理由を、レビュアーが既に見ている場所 — PRの説明文 — に書くだけです。
- **ソースコードをどこかに読み込んだり送信したりしません。** アカウントも、アップロードも、テレメトリもありません。ブロックは、あなたが既に動かしているエージェントによって、あなた自身のモデルのサブスクリプション上で、あなたのマシン上で作成されます。
- **前向き専用です。** これから開かれるPRに理由を書きます。閉じた過去を書き換えることは決してなく、何か問題が起きてもgitコマンドには触れません — フックのエラーは常にフェイルオープンします。

**無料・MIT、ずっと。** [Backthread](https://backthread.dev)（有料のホスト型レイヤー）は、
ローカルのフックが構造上できないこと — チーム横断、履歴、そしてプロアクティブなプッシュ — を
担います。理由があなたに届けられ、コードベース全体を横断して、全員のエージェントを横断して検索
できるようになります。このフックは、それ自体で役立つのにそのどれも必要としません。チームビュー
が欲しくなったら、[backthread.dev](https://backthread.dev) にあります。

## 仕組み

フックは2つの瞬間を監視します。PRを開くとき（`gh pr create`）と、デフォルトブランチへ直接
コミットするときです。理由ブロックのないものを見つけると、エージェントに対して、そのセッション
自身の推論から根拠のある前向きなブロック — 変更の背後にある **決定、トレードオフ、前提、制約** —
を作成し、コマンドを再実行するよう促します。ブロックは不可視のマーカーで囲まれるため、一度だけ
書き込まれ、重複することはありません。

- **フィーチャーブランチのコミットはPRに委ねます。** 複数のセッションにまたがる作業はローカルで
  引き継がれるため、たとえ別のセッションがPRを開いても、そのブロックはブランチ全体をカバーします。
- **何かがでっち上げられることはありません。** エージェントはまず簡単なセルフチェックを行い、
  実際の決定にたどれない行はすべて捨てます。セッションが検討しなかったなら、ブロックは追加され
  ません。
- **gitコマンドを決してブロックしません。** あらゆる失敗モードは静かなノーオペです — 最悪でも、
  ブロックが追加されないだけです。

各ブロックには小さな可視の帰属表示が付いているので、レビュアーはそれがどこから来たかを確認でき、
自由に編集・削除できます。

## 制御

- **リポジトリごとにオフにする:** `git config add-reasoning-to-prs.disabled true`
- **単一のコミット/PRをスキップする:** コマンドのどこかに `[skip-why]` を入れます。
- **グローバルにオフにする:** Claude Codeを起動する環境で `ADD_REASONING_TO_PRS_DISABLE=1`
  を設定します。

## ロードマップ

プロジェクトの正直な現状 — 何が動作し、次は何で、まだ何をしないのか。

- **今日動作するもの:** `gh pr create` 時の「理由」ブロック、直接プッシュ時のコミットメッセージ
  へのフォールバック、ブランチをまたぐマルチセッションの引き継ぎ、100%ローカル（あなた自身の
  モデル、アカウント不要）、でっち上げなし、そしてフェイルオープン。
- **次に来るもの:** Cursor と Codex のサポート（現時点ではClaude Codeのみ — これがリストの
  最上位です）· ブラウザで開かれたPRへの対応 · よりタイトな理由ブロックの書式。
- **既知のギャップ:** マルチセッションの収集はベストエフォート（ローカルのブランチごとの
  スクラッチパッド）であり、スカッシュマージ後にブロックがどう読めるかはまだ調整中です。

完全なリストは [Issues](https://github.com/backthread/add-reasoning-to-prs/issues) と [Discussions](https://github.com/backthread/add-reasoning-to-prs/discussions) にあります —
あなたにとって重要な制約に 👍 を付ければ、リストの上位に上がっていきます。

## コントリビュート

コントリビューションを歓迎します — 特にバグ修正、より鋭いプロンプト/ガイダンス文言、エッジ
ケースへの対応、そしてより多くのエージェントのサポートです。これは小さく単一目的のツールで、
その姿勢を保つつもりなので、スコープを広げる要望は通常（丁寧に、理由を添えて）お断りしています。
まずは[コントリビューションガイド](../../CONTRIBUTING.md)と [`good first issue`](https://github.com/backthread/add-reasoning-to-prs/labels/good%20first%20issue)
ラベルから始めてください。アイデアが合うか分からない場合は、まず [Discussion](https://github.com/backthread/add-reasoning-to-prs/discussions)
を開いてください。
